## 2.1 Personas

For each user type (primary, secondary, admin if applicable):

```
**Persona: {Name}**
- Role: {who they are}
- Goal: {what they want to achieve}
- Pain point: {what frustrates them today}
- Tech level: {beginner | intermediate | advanced}
```

> "Who are the main users? Describe the primary person who'll use this daily."

One persona at a time. Ask if there are secondary users after the primary is defined.

## 2.2 User Journeys

For each persona, map 1-2 key journeys:

```
**Journey: {Action name}**
1. {User does X} → {System responds Y}
2. {User does X} → {System responds Y}
3. ...
Outcome: {What the user achieved}
Emotion: {How they feel at the end}
```

> "Walk me through what [persona] does when they [core action]. Step by step."

## 2.3 Success Criteria

Define measurable success for:

**User success:**
- "How will users know the product works for them?"

**Business success:**
- "What metric tells you the product is succeeding?" (users, revenue, retention, etc.)

**Technical success:**
- "What performance/reliability bar must be met?"

```
| Criteria | Metric | Target |
|----------|--------|--------|
| {User adoption} | {MAU} | {X users in Y months} |
| ... | ... | ... |
```
